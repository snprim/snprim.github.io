title: R vs. other programming languages/softwares
---

Besides **R**, I have learned **Python**, **Java**, and **SAS**. Obviously, each of these languages has its own pros and cons. I think R and Python are easier to start using than Java and SAS. The syntax of R and Python are intuitive, and both do not require a lot of specifications. For example, Java requires users to declare a new object and identify its data type—“int,” “double,” “char,” etc.—but, in R and Python, we can simply declare a new object and give it a value at the same time. While SAS does not require the extra steps required by Java, the syntax is not as similar to natural languages; also, it sometimes deals with codes in unexpected ways that require the users to know exactly how SAS works behind the scene to avoid bugs. For these reasons, it took me longer to get used to writing Java and SAS codes than to writing R and Python codes, both of which have a lot of automatic processing behind the scene and therefore easier to pick up.

In terms of R, it is nice to have all kinds of packages that can be easily installed. Almost every data analysis can be done with a certain package. R is also great for making publication-ready plots. Furthermore, some R packages come with data sets. Especially for data analysis, R is a flexible and easy-to-use tool.

Indeed, some features of other programming languages are missed in R. Although Java might be hard to learn because it requires the users to know more about the specs, the extra details make debugging easier. Also, Java emphasizes the idea of object oriented programming, and thus the codes are structured and easy to understand. Java also has an important feature, garbage collection, that frees up memory automatically and is a fast language. In terms of Python, it seems a better option to connect to online sources, whether for using APIs or webscraping. Memory, speed, and webscraping are not yet something we need while using R, so I can’t quite say I “miss” Java and Python for these reasons, but these features could definitely make Java and Python better options than R for certain tasks.

Overall, I consider R easy to learn, at least in the beginning. Every language can be difficult when the tasks become complex. But, for a beginner, it is easy to type in a few lines in R and, voila!, a nice-looking plot has been created.
